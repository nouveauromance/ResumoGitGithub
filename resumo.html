Resumo - Git & Github

Como aplicar no dia à dia, ideias e conceitos gerais

Ideia geral:

O Git é um sistema de controle de versões de arquivos responsável por armazenar um histórico
de alterações e versões de determinado projeto.

O Github é uma plataforma que atua como rede social para profissionais da área de TI, programadores
e pessoas com ramo. Serve para o compartilhamento de projetos, repositórios e materiais.

Vejamos, a seguir, os principais conceitos e comandos do Git:

Como o sistema funciona?

O sistema armazena os arquivos em repositórios, que funcionam como pastas que ficam anexadas ao seu perfil.
Nelas, você realiza os chamados commits, que funcionam, basicamente, como checkpoints (salvamentos) no seu código.
São registros que incluem as mudanças nos arquivos e exibem uma mensagem explicando o que foi modificado.

Repositórios e Trees

  • Um repositório é como uma pasta ou diretório que contém todos os arquivos e o histórico de um projeto.
    Cada commit é uma entrada no histórico do repositório que contém informações sobre as alterações feitas.
  • Árvores (trees) representam a estrutura do diretório e arquivos em um commit específico, que tem como função registrar
    a organização do projeto     ao longo do histórico de desenvolvimento.

Commit e o ato de Comitar

1. Preparação:
Antes de comitar, você usa "git add" para adicionar as alterações que deseja incluir no commit.
2. Comitando:
Depois, o comando "git commit" captura um snapshot dessas alterações, salvando-as no histórico do projeto.
3. Mensagem:
Junto com as mudanças, você inclui uma mensagem descritiva (com -m "sua mensagem") para explicar o que foi alterado.

Vantagens de comitar

  • Rastreamento:
    Permite ver o histórico de alterações, quem fez e quando.
  • Backups:
    Facilita o retorno a versões anteriores do código.
  • Colaboração:
    Ideal para trabalhar em equipe, pois cada commit mostra as contribuições individuais.
  • Organização:
    Ajuda a manter o código organizado e com um histórico claro.

Branches & Merges

  • As “ramificações”, ou branches, permitem que você crie linhas separadas para trabalhar em alterações
    sem afetar a linha principal do projeto.
  • Cada branch é uma ramificação independente do código-fonte, possibilitando que você isole e desenvolva novas
    funcionalidades, refatore o código ou faça correções sem interferir no código existente na
    branch principal, que geralmente é nomeada como "main".
  • Em um projeto com branches diferentes, a fusão, ou merge, permite combinar as alterações de volta à
    linha principal, quando as alterações estão prontas.

Repositórios Remotos

Um repositório remoto é uma versão de um repositório que está hospedada em um servidor remoto,
como o GitHub, GitLab, ou outro serviço similar, ou mesmo em um servidor interno da sua organização. Ele serve como um
local centralizado para armazenar e compartilhar o código-fonte de um projeto, permitindo que vários desenvolvedores
colaborem e sincronizem suas alterações.

O que um repositório remoto faz?

  • Armazenamento centralizado:
    Fornece um local seguro e centralizado para armazenar o código-fonte do projeto.
  • Colaboração:
    Permite que múltiplos desenvolvedores trabalhem no mesmo projeto, sincronizando suas alterações e evitando conflitos.
  • Controle de versão:
    Mantém um histórico completo de todas as alterações feitas no código, permitindo que os desenvolvedores voltem a versões anteriores, se necessário.
  • Compartilhamento:
    Facilita o compartilhamento do código com outros desenvolvedores, seja para colaboração ou para disponibilização do projeto para download.

Push & Pull

Em Git, "push" e "pull" são comandos usados para sincronizar o repositório local com um repositório remoto.
Push envia as alterações do seu repositório local para o repositório remoto, enquanto Pull traz as alterações do
repositório remoto para o seu repositório local.

Push

  • O que faz:
    Envia as alterações feitas no seu repositório local para o repositório remoto.
  • Quando usar:
    Quando você deseja compartilhar suas alterações com outros colaboradores ou atualizar o repositório remoto com suas modificações.
  • Como usar:
    git push origin nome_da_branch.
